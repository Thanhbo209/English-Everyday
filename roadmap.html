Two Roles, One Classroom

Everything flows between the teacher who creates, and the students who learn

👩‍🏫

Teacher

CLASSROOM OWNER

Creates the classroom, builds all vocabulary content, assigns activities to students, and can launch live multiplayer game sessions in real time.

  • Create classroom & generate a 6-char join code
  • Build vocabulary sets (term, definition, audio, image)
  • Choose from 51+ activity types to assign as homework
  • Set due dates and view completion/score stats
  • Launch a live game session — students join instantly
  • View per-student progress and export reports
🎓

Student

CLASSROOM MEMBER

Joins the classroom with a code. Sees their personal task list, studies using any available activity type, and joins live games the teacher launches.

  • Join classroom with a 6-digit code
  • See task list with due dates and status
  • Study vocabulary through any assigned activity
  • Free-practice mode using any unlocked activity
  • Join live teacher-launched game sessions
  • View own XP, streak, and leaderboard rank

Architecture

Single server — simple, internal-grade, easy to host and maintain

Tech Stack

Frontend
React + Vite · React Router · Vanilla CSS · Web Audio API (for audio activities)
API Server
Node.js + Fastify · REST endpoints + WebSocket (ws) for live sessions
Database
PostgreSQL + Prisma ORM · activity_type enum covers all 51+ types
Media Storage
MinIO (local S3-compatible) for vocab images + audio files · served via signed URLs
Auth
JWT (1hr access) · bcrypt passwords · role embedded in token payload
Deployment
Single VPS · Docker Compose · Nginx reverse proxy · SSL via Let's Encrypt

model VocabItem {
  id              String   @id @default(uuid())
  set             VocabSet @relation(fields:[setId], references:[id])
  setId           String
  term            String              // the vocabulary word
  definition      String              // meaning / translation
  phonetic        String?             // /fəˈnetɪk/
  partOfSpeech    String?             // noun, verb, adj…
  exampleSentence String?
  imageUrl        String?             // MinIO object URL
  audioUrl        String?             // MinIO object URL
  orderIndex      Int      @default(0)
  masteries       VocabMastery[]
}

model Assignment {
  id           String       @id @default(uuid())
  classroom    Classroom    @relation(fields:[classroomId], references:[id])
  classroomId  String
  teacherId    String
  title        String
  activityType String       // one of 51 enum values e.g. "Q3", "G6", "BATTLE"
  vocabSet     VocabSet     @relation(fields:[vocabSetId], references:[id])
  vocabSetId   String
  config       Json         @default("{}")  // time limit, question count, etc.
  dueAt        DateTime?
  createdAt    DateTime     @default(now())
  submissions  Submission[]
}

model VocabMastery {
  student      User      @relation(fields:[studentId], references:[id])
  studentId    String
  vocabItem    VocabItem @relation(fields:[vocabItemId], references:[id])
  vocabItemId  String
  status       String    @default("new")  // new | learning | known
  attempts     Int       @default(0)
  lastSeenAt   DateTime  @default(now())
  @@id([studentId, vocabItemId])
}

model LiveSession {
  id           String   @id @default(uuid())
  classroom    Classroom @relation(fields:[classroomId], references:[id])
  classroomId  String
  teacherId    String
  activityType String   // same enum values
  vocabSetId   String
  config       Json     @default("{}")
  pin          String   @unique  // 6-char display code for students
  status       String   @default("waiting") // waiting|active|finished
  scores       Json     @default("{}")
  startedAt    DateTime @default(now())
  endedAt      DateTime?
  players      LiveSessionPlayer[]
}

Word Search Generator

The server generates the word search grid on-the-fly from the vocab set. Algorithm places words in all 8 directions then fills remaining cells with random letters.

// API: GET /activities/W1/:vocabSetId/grid
// Returns:
{
  grid: string[][],     // 15×15 letter grid
  words: [
    { word: "apple", clue: "a round fruit",
      found: false }
  ]
}
Matching Algorithm (M1, M2, O1, O2)
  • Sample 6–10 items from vocab set per round
  • Shuffle both columns independently
  • Correct pair → lock with animation + points
  • Wrong pair → brief shake; no penalty points
  • All pairs matched → round complete, new round begins

Scope Reminder

This is an internal classroom tool. No multi-tenancy. No billing. No public registration. One teacher creates a classroom, shares a code, and their students join. The platform's only job is to make vocabulary learning engaging through 51+ activity types. Build Phase 1 → 2 → 3 in sequence. Don't skip to Phase 8 before the basics work.
